디테일은 날리고, 핵심만.
- **메커니즘의 분리 가능성:**
    연구진은 예시 데이터($S$)로부터 태스크 벡터를 추출한 뒤, 모델이 새로운 질문($x$)을 처리할 때 예시 데이터를 모두 지우고 추출한 벡터만 주입(Inject)해 보았습니다. 그 결과, 예시가 있을 때와 거의 동일한 수준의 성능을 보였습니다. 이는 모델이 실제로 정보를 벡터 형태로 압축해 사용함을 시사합니다.
    
- **태스크 벡터의 범용성:**
    동일한 작업(예: 한영 번역)에 대해 서로 다른 예시 세트를 주어도 생성되는 태스크 벡터들은 서로 매우 유사하며 공간상에서 군집(Clustering)을 이룹니다.
    
- **모델 거동 제어 (Steering):**
    특정 작업의 벡터를 모델에 강제로 주입하면, 입력된 예시와 상관없이 해당 작업을 수행하게 만들 수 있습니다. 예를 들어, '대문자 변환' 예시를 주고 '번역 벡터'를 주입하면 모델은 예시를 무시하고 번역을 수행합니다.

# 결론
초기 레이어들이 컨텍스트 내의 예시들을 하나의 task vector로 압축하고 마지막 예시 토큰의 activation에 넣으면, 뒤 레이어들은 예시 참조 없이 이 토큰에서 task vector만 꺼내 읽고 작업한다.

### task vector 활용
- **벡터의 덧셈 (Composition):** 두 가지 다른 작업의 벡터를 더해서 새로운 능력을 부여할 수 있습니다.
    
    - 예: `(프랑스어 번역 벡터) + (격식 있는 말투 벡터) = 격식 있는 프랑스어 번역`
        
    - 이것은 모델이 여러 제약 조건을 동시에 만족해야 하는 복잡한 지시사항을 벡터 수준에서 결합할 수 있음을 의미합니다.
        
- **벡터의 뺄셈 (Analogy & Removal):** 특정 성향을 제거하는 것도 가능합니다.
    
    - 예: `(독성이 포함된 응답 벡터) - (독성 성분 벡터) = 정제된 응답`
        
    - 워드 임베딩(Word2Vec)에서 `King - Man + Woman = Queen`이 성립하듯, 태스크 수준에서도 모델의 특정 행동 양식을 더하거나 뺄 수 있습니다.
        
- **평균화 (Averaging for Robustness):** 하나의 예시 세트에서 추출한 벡터는 편향(Bias)이 있을 수 있습니다. 연구자들은 서로 다른 예시 세트(S1, S2, S3...)에서 추출한 여러 태스크 벡터의 평균을 내어 사용하면, 훨씬 더 범용적이고 강력한 '골드 태스크 벡터'를 얻을 수 있음을 발견했습니다.
