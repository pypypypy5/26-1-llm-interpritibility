# 문제
기존 ([[Activation Patching]])에는 모델의 특정 '레이어'나 '뉴런'이 중요하다고 추측하는 수준이었으나, 
1. 정보가 어떤 흐름을 거쳐 결과에 도달하는지 정확히 파악하기 어려웠음.
2. 노드를 통째로 갈아끼워서 노드가 이후에 미치는 영향 한번에 변경 - 분석 뭉뚱그려짐

# method
>_**계산 그래프에서 특정 엣지를 망가트리고 최종 출력에 얼마나 영향이 가는지를 측정하자.
>$\rightarrow$ 결과가 많이 망가지면 중요한 연결, 덜 망가지면 안 중요한 연결_**

1. 두 가지 입력 준비: 
	- 깨끗한 입력(Clean Input): 모델이 정답을 맞히는 정상적인 데이터 (예: "The capital of France is Paris").
    
    - **오염된 입력(Corrupted Input):** 모델이 혼란을 느끼게 하는 데이터 (예: "The capital of Germany is Berlin").
        
2. **엣지 차단(Intervention):** 특정 노드 A에서 노드 B로 가는 정보(엣지)를 '오염된 입력'을 넣었을 때의 값으로 강제로 교체(Patching).
    
3. **영향력 측정:** 이렇게 정보를 바꿨을 때, 최종 출력값(Logits)이 정답에서 얼마나 멀어지는지 확인합니다.

# 평가
## 성과
### 주요 기여
- **가설 검증 프레임워크:** "이 동작은 A 노드에서 B 노드로 가는 경로 때문에 발생한다"는 가설을 세우고 이를 수치적으로 테스트할 수 있는 엄격한 방법론을 제시했습니다.

- **인덕션 헤드(Induction Heads)의 정교화:** 이전 연구에서 발견된 '패턴 반복'을 담당하는 인덕션 헤드가 실제로 어떤 경로를 통해 정보를 전달하는지 더 구체적으로 밝혀냈습니다.

- **GPT-2의 행동 분석:** GPT-2 모델을 대상으로 실제 실험을 진행하여, 특정 문법적/논리적 구조를 처리할 때 활성화되는 구체적인 계산 경로(Circuit)를 식별했습니다.

- **오픈소스 라이브러리:** 다른 연구자들이 자신의 모델에 이 기법을 쉽게 적용할 수 있도록 효율적인 실험 프레임워크를 공개했습니다.
