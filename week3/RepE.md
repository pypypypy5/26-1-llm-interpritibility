= Representation Engineering
# 문제
각 뉴런이 어떤 역할 하는지 하나하나 잡아내는건 어렵다.
# method
뉴런 단위로 보는게 아니라, 내부 활성화 상태를 뜯어서 high level로 나타나는 특징만 살펴보자. allign 등에 쓰려면 그게 더 실용적이다.

### Representation Reading

모델의 내부 상태에서 특정 개념(예: 정직함, 유해함, 감정 등)이 어떻게 발현되는지 찾아내기.

- **대조적 자극(Contrastive Stimuli):** 특정 개념이 포함된 문장과 포함되지 않은 문장(예: "사실을 말해줘" vs "거짓말을 해줘")의 쌍을 모델에 입력합니다.
    
- **활성화 추출:** 각 층(Layer)에서 발생하는 뉴런들의 활성화 값(Hidden States)을 수집합니다.
    
- **선형 적대적 훈련(Linear Adversarial Training, [[LAT]]) 및 [[PCA]]:** * 수집된 고차원 벡터 데이터에서 두 상태(정직 vs 거짓)를 가장 잘 구분해내는 **'방향(Direction)'**을 찾습니다.
    
    - 이때 **PCA(주성분 분석)**를 통해 데이터의 가장 큰 변동성을 찾거나, **LAT**를 통해 특정 개념을 식별하는 '읽기 벡터(Reading Vector)'를 추출합니다.
        
    - 결과적으로 특정 개념을 대표하는 하나의 **'개념 벡터'**를 얻게 됩니다.

### Representation Control

읽기 단계에서 찾아낸 '개념 벡터'를 사용하여 모델의 추론 과정을 실시간으로 조작하는 단계입니다.

- **읽기 벡터의 적용:** 모델이 새로운 입력을 처리할 때, 특정 레이어의 활성화 값에 앞서 찾은 '개념 벡터'를 더하거나 뺍니다.
    
    - **식:** $h' = h + \alpha v$ (여기서 $h$는 원래 활성화, $v$는 개념 벡터, $\alpha$는 강도 조절 계수)
        
- **행동 유도:**
    
    - **정직성 강화:** 모델의 내부 상태를 '정직' 방향으로 밀어주면, 모델은 학습된 지식에 더 충실한 답변을 내놓습니다.
        
    - **유해성 억제:** '유해함'을 나타내는 벡터를 반대 방향으로 빼주면, 프롬프트 엔지니어링 없이도 유해한 답변을 피하게 됩니다.

# 평가
## 성과
1. top-down 식 접근을 처음 시도
2. 실제로 정렬에 쓸 수 있도록 feature 추출하고 적용하는데 성공

## 한계
1. 내부 활성화 값이 선형이라 가정하지만, 실제로는 비선형, 더 복잡할 가능성 높음
2. 어느 레이어에서 추출하고, 조작 강도를 어떻게 설정할지를 정하는 과정은 정형화 안됨. 그냥 휴리스틱과 시행착오.
